# 微应用开发规范

## 1. 微应用概念

微应用是一种独立的应用程序，它可以：
- **独立运行**：作为单独的应用程序运行
- **嵌入运行**：在智能家居控制面板的容器环境中运行
- **模块化**：具有明确的边界和接口
- **可插拔**：可以轻松添加、移除或替换

## 2. 微应用目录结构

每个微应用应遵循以下目录结构：

```
apps/
  应用名称/
    index.html          # 应用入口文件
    assets/             # 资源文件
      images/           # 图片
      icons/            # 图标
    js/                 # JavaScript文件
    css/                # 样式文件
    config/             # 应用配置
    README.md           # 应用说明文档
```

### 2.1 目录说明

- **index.html**：应用的主入口文件，必须存在
- **assets/**：存放应用所需的静态资源
- **js/**：存放应用的JavaScript代码
- **css/**：存放应用的样式文件
- **config/**：存放应用的配置文件
- **README.md**：应用的说明文档

## 3. 微应用配置

### 3.1 应用配置文件

在 `apps.json` 中添加微应用配置：

```json
{
  "id": "app1",
  "name": "应用名称",
  "icon": "图标",
  "path": "应用路径"
}
```

- **id**：应用的唯一标识符，建议使用小写字母和下划线
- **name**：应用的显示名称
- **icon**：应用的图标，可以是emoji或SVG路径
  - Emoji格式：`📒`
  - SVG格式：`svg:应用名称/icon.svg`
- **path**：应用的入口文件路径

### 3.2 应用内部配置

每个微应用可以在其目录下创建 `config/config.json` 文件，用于存储应用特定的配置：

```json
{
  "name": "应用名称",
  "version": "1.0.0",
  "description": "应用描述",
  "author": "开发者",
  "settings": {
    "key1": "value1",
    "key2": "value2"
  }
}
```

## 4. 微应用与容器通信

### 4.1 通信方式

微应用与容器之间通过 `postMessage` API进行通信。

### 4.2 容器向微应用发送的消息

#### 4.2.1 初始化消息

```javascript
// 容器发送
iframe.contentWindow.postMessage({
  type: 'app-container-init'
}, '*');

// 微应用接收
window.addEventListener('message', function(event) {
  if (event.data && event.data.type === 'app-container-init') {
    // 容器初始化完成
    // 可以进行应用初始化操作
  }
});
```

#### 4.2.2 菜单点击消息

```javascript
// 容器发送
iframe.contentWindow.postMessage({
  type: 'menu-click'
}, '*');

// 微应用接收
window.addEventListener('message', function(event) {
  if (event.data && event.data.type === 'menu-click') {
    // 处理菜单点击事件
  }
});
```

### 4.3 微应用向容器发送的消息

#### 4.3.1 设置应用名称

```javascript
// 微应用发送
window.parent.postMessage({
  type: 'set-app-name',
  name: '应用名称'
}, '*');
```

#### 4.3.2 菜单操作消息

```javascript
// 微应用发送
window.parent.postMessage({
  type: 'menu-action',
  data: 'toggle-menu'
}, '*');
```

### 4.4 通信接口规范

| 消息类型 | 方向 | 描述 | 数据格式 |
|---------|------|------|---------|
| app-container-init | 容器 → 微应用 | 容器初始化完成 | `{ type: 'app-container-init' }` |
| menu-click | 容器 → 微应用 | 菜单被点击 | `{ type: 'menu-click' }` |
| set-app-name | 微应用 → 容器 | 设置应用名称 | `{ type: 'set-app-name', name: '应用名称' }` |
| menu-action | 微应用 → 容器 | 菜单操作 | `{ type: 'menu-action', data: '操作类型' }` |

## 5. 微应用独立运行

### 5.1 独立运行模式

微应用应支持独立运行，通过检查是否在容器环境中来调整行为：

```javascript
// 检查是否在容器中
const isInContainer = window.self !== window.top;

if (isInContainer) {
  // 在容器中运行
  // 初始化与容器的通信
} else {
  // 独立运行
  // 初始化独立运行的UI和功能
}
```

### 5.2 独立运行的UI适配

当微应用独立运行时，应提供完整的UI：
- 顶部导航栏
- 底部导航栏
- 适当的响应式设计
- 完整的错误处理

## 6. 微应用开发流程

### 6.1 创建微应用

1. **创建目录**：在 `apps/` 下创建应用目录
2. **创建入口文件**：创建 `index.html` 文件
3. **配置应用**：在 `apps.json` 中添加应用配置
4. **开发功能**：实现应用的核心功能
5. **测试运行**：测试独立运行和容器中运行
6. **编写文档**：创建 `README.md` 文件

### 6.2 开发工具

- **编辑器**：推荐使用VS Code
- **浏览器**：推荐使用Chrome进行调试
- **本地服务器**：使用 `python -m http.server 8000` 启动本地服务器

### 6.3 测试流程

1. **独立运行测试**：直接访问应用的 `index.html` 文件
2. **容器运行测试**：通过智能家居控制面板访问应用
3. **功能测试**：测试应用的各项功能
4. **兼容性测试**：测试在不同浏览器中的表现

## 7. 微应用最佳实践

### 7.1 代码组织

- **模块化**：将代码拆分为多个模块
- **组件化**：将UI拆分为可复用的组件
- **关注点分离**：将业务逻辑与UI分离

### 7.2 性能优化

- **资源加载**：优化资源加载顺序
- **代码压缩**：压缩HTML、CSS和JavaScript
- **缓存策略**：合理使用缓存
- **懒加载**：按需加载资源

### 7.3 安全性

- **输入验证**：验证所有用户输入
- **XSS防护**：防止跨站脚本攻击
- **CSP**：使用内容安全策略
- **权限控制**：合理设置应用权限

### 7.4 可访问性

- **语义化HTML**：使用语义化的HTML标签
- **ARIA属性**：添加适当的ARIA属性
- **键盘导航**：支持键盘导航
- **屏幕阅读器**：确保屏幕阅读器兼容

## 8. 微应用示例

### 8.1 基本结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>微应用示例</title>
    <style>
        /* 样式 */
    </style>
</head>
<body>
    <!-- 应用内容 -->
    <div id="app">
        <h1>微应用示例</h1>
        <p>这是一个微应用示例</p>
    </div>

    <script>
        // 检查是否在容器中
        const isInContainer = window.self !== window.top;

        // 初始化函数
        function init() {
            if (isInContainer) {
                // 在容器中运行
                initContainerMode();
            } else {
                // 独立运行
                initStandaloneMode();
            }
        }

        // 容器模式初始化
        function initContainerMode() {
            // 发送应用名称
            window.parent.postMessage({
                type: 'set-app-name',
                name: '微应用示例'
            }, '*');

            // 监听容器消息
            window.addEventListener('message', handleContainerMessage);
        }

        // 独立模式初始化
        function initStandaloneMode() {
            // 添加独立运行的UI
            const app = document.getElementById('app');
            const header = document.createElement('header');
            header.innerHTML = '<h1>微应用示例（独立运行）</h1>';
            app.insertBefore(header, app.firstChild);
        }

        // 处理容器消息
        function handleContainerMessage(event) {
            if (event.data && event.data.type) {
                switch (event.data.type) {
                    case 'app-container-init':
                        // 容器初始化完成
                        console.log('容器初始化完成');
                        break;
                    case 'menu-click':
                        // 菜单被点击
                        console.log('菜单被点击');
                        break;
                }
            }
        }

        // 初始化应用
        init();
    </script>
</body>
</html>
```

### 8.2 应用配置示例

```json

  "id": "example-app",
  "name": "示例应用",
  "icon": "📱",
  "path": "example/index.html"
}
```

## 9. 常见问题处理

### 9.1 应用加载失败

- 检查应用路径是否正确
- 检查应用文件是否存在
- 检查应用文件是否有语法错误

### 9.2 通信失败

- 检查 `postMessage` 的使用是否正确
- 检查消息类型是否匹配
- 检查消息数据格式是否正确

### 9.3 样式冲突

- 使用CSS命名空间
- 使用BEM命名规范
- 避免使用全局样式

### 9.4 性能问题

- 优化资源加载
- 减少DOM操作
- 使用防抖和节流

## 10. 总结

本规范旨在指导微应用的开发，确保微应用能够：
- 独立运行
- 在容器中运行
- 与容器正确通信
- 具有良好的性能和安全性

随着项目的发展，本规范将不断更新和完善。
